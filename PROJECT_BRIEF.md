# 대신 결정해드립니다 (대결)

## 프로젝트 소개

`대결`은 여러 선택지 사이에서 결정을 내리기 어려운 사용자를 위한 개인화 추천 웹앱이다.

사용자가 고민과 선택지를 입력하면, 앱은 저장된 사용자 성향과 현재 상황을 바탕으로 하나를 골라 짧은 이유와 함께 추천한다.

## 해결하려는 문제

사용자는 여러 선택지가 주어졌을 때 무엇을 골라야 할지 판단하지 못하고 망설인다.

예를 들면 다음과 같다.

- 구매하려는 상품 중 어떤 제품을 선택해야 할지 결정하지 못한다.
- 점심 메뉴를 고르지 못한다.
- 여행지 후보 중 어디를 방문할지 결정하지 못한다.

이러한 일상적인 고민에도 시간과 에너지가 소모된다. `대결`은 개인화된 추천을 통해 사용자의 결정 부담을 줄인다.

## 핵심 사용자

상품, 음식, 여행지 등 일상적인 선택을 내리는 데 어려움을 느끼는 사람

## 첫 배포 버전의 핵심 흐름

1. 사용자가 회원가입하고 로그인한다.
2. 사전 질문에 답하여 자신의 성향 정보를 등록한다.
3. 사용자가 현재 고민과 두 개 이상의 선택지를 입력한다.
4. 저장된 성향과 현재 입력만으로 추천하기 어렵다면 앱이 추가 질문을 한다.
5. 앱이 선택지 하나를 이유와 함께 추천한다.
6. 사용자가 추천을 채택하거나 다시 추천을 요청한다.
7. 앱이 추천 결과와 사용자 반응을 저장한다.

## 추천 전 추가 질문

앱은 필요한 경우에만 분야에 맞는 추가 질문을 한다.

- 최대 질문 수: 5개
- 질문에 활용할 수 있는 정보:
  - 최근에 내렸던 유사한 결정
  - 가격 또는 예산
  - 중요하게 생각하는 기능과 성능
  - 디자인 취향
  - 현재 상황과 우선순위

## 추천 결과 형식

추천 결과에는 다음 내용을 표시한다.

1. 추천한 선택지 하나
2. 추천 이유에 대한 짧은 설명
3. 나머지 선택지를 고르지 않은 이유에 대한 짧은 설명
4. 추천 확신도 백분율

예시:

```text
추천: A 식당

추천 이유:
오늘은 가볍게 먹고 싶다는 답변과 예산을 고려했습니다.

다른 선택지를 고르지 않은 이유:
- B 식당: 예산을 초과합니다.
- C 식당: 최근에 비슷한 메뉴를 먹었습니다.

추천 확신도: 82%
```

## 저장할 정보

- 회원가입 이메일
- 사전 성향 질문의 답변
- 과거의 고민과 입력한 선택지
- 이전 추천 결과
- 추천에 대한 사용자 반응
- 추천 채택률

## 채택률 계산 규칙

사용자가 `이 추천이 마음에 들어요` 버튼을 누르면 해당 추천을 채택한 것으로 처리한다.

사용자가 `다시 추천해주세요` 버튼을 누르면 해당 추천을 채택하지 않은 것으로 처리한다.

```text
채택률 = 채택한 추천 수 ÷ 전체 추천 수 × 100
```

## 첫 배포 버전에 포함할 기능

- 회원가입 및 로그인
- 사전 성향 질문
- 사용자 성향 정보 저장
- 자유로운 분야의 고민 입력
- 두 개 이상의 선택지 입력
- 필요한 경우 최대 5개의 추가 질문
- 개인화된 추천
- 추천 이유와 제외 이유
- 추천 확신도
- 추천 채택 및 다시 추천 버튼
- 과거 추천 결과와 채택률 저장

## 첫 배포 버전에서 제외할 기능

- 상품 가격 자동 비교
- 식당 자동 검색
- 여행지 정보 자동 수집
- 외부 서비스 연동
- 분야별 전용 추천 엔진
- 네이티브 모바일 앱

첫 버전은 반응형 웹앱으로 제작한다. 모든 분야를 지원하지만 사용자가 고민과 선택지를 직접 입력하는 공통 흐름을 사용한다.

## 첫 배포 버전의 성공 기준

- 사용자가 회원가입 후 성향 질문에 답할 수 있다.
- 사용자가 두 개 이상의 선택지를 입력할 수 있다.
- 추천에 필요한 정보가 부족하면 최대 5개의 추가 질문을 받을 수 있다.
- 사용자가 추천 결과, 짧은 추천 이유, 제외 이유, 확신도를 확인할 수 있다.
- 사용자가 추천을 채택하거나 다시 추천받을 수 있다.
- 이전 추천 결과와 채택률이 저장된다.

## 기술 선택

첫 MVP는 `Next.js + Supabase`를 사용한다.

- Next.js: 화면 구성, 입력 폼, 추천 결과 표시, 반응형 웹앱 구현
- Supabase: 회원가입, 로그인, PostgreSQL 데이터 저장
- OpenAI Responses API: 개인화 추천 결과 생성

기술 선택 이유와 tradeoff는 `TECH_DECISION.md`에 기록한다.
