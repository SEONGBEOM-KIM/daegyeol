# M1 첫 화면 검증 체크리스트

## 목적

첫 화면이 사용자의 첫 방문에서 앱의 목적을 이해시키는지 반복 가능하게 확인한다.

## 자동 검증

다음 명령어를 실행한다.

```bash
npm run test:m1
npm run build
```

자동 검증에서 확인하는 항목:

- 앱 이름 `대신 결정해드립니다`가 코드에 있다.
- 앱 목적을 설명하는 문장이 있다.
- `시작하기` 버튼이 있다.
- 사용 흐름 3단계가 있다.
- 모바일 화면을 위한 반응형 스타일이 있다.

## 수동 검증

개발 서버를 실행한다.

```bash
npm run dev
```

브라우저에서 접속한다.

```text
http://localhost:3000
```

확인 항목:

- [ ] 첫 화면에 `대신 결정해드립니다`가 보인다.
- [ ] 앱이 무엇을 해주는지 한 문장으로 이해된다.
- [ ] `시작하기` 버튼이 보인다.
- [ ] `어떻게 도와주나요?` 버튼이 보인다.
- [ ] `고민 입력 → 성향 반영 → 하나 추천` 흐름이 보인다.
- [ ] 브라우저 폭을 줄여도 카드가 세로로 쌓이고 내용이 잘리지 않는다.

## 완료 기준

- 자동 검증이 통과한다.
- 브라우저에서 첫 화면 목적을 바로 이해할 수 있다.
- 모바일 크기에서도 핵심 내용이 보인다.
