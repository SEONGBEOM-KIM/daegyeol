# Day 7 Debug Notes

## 학습 목표

오류를 만나면 바로 코드를 고치기보다, 재현 방법과 오류 메시지를 먼저 정리하고 가장 작은 수정으로 해결한다.

## 재현 단계

1. `app/page.tsx`에서 첫 번째 카드 제목 태그를 일부러 잘못 닫았다.
2. 다음 명령어를 실행했다.

```bash
npm run build
```

## 예상 결과

Next.js 빌드가 성공해야 한다.

## 실제 결과

빌드가 실패했다.

```text
Expected '</', got 'jsx text'
./app/page.tsx:24:33
<h2>고민 입력</h3>
```

## 원인

JSX 태그는 여는 태그와 닫는 태그가 정확히 일치해야 한다.

잘못된 코드:

```tsx
<h2>고민 입력</h3>
```

`h2`로 열었는데 `h3`로 닫았기 때문에 파서가 다음 텍스트와 태그 구조를 이해하지 못했다.

## 최소 수정

닫는 태그만 `h2`로 고쳤다.

```tsx
<h2>고민 입력</h2>
```

## 배운 점

- 오류 메시지에는 파일명, 줄 번호, 문제가 된 코드 조각이 들어 있다.
- JSX 오류는 대부분 태그 짝, 괄호, 따옴표, 중괄호 위치를 먼저 확인한다.
- 원인을 모를 때는 큰 리팩터링을 하지 말고, 오류가 가리키는 가장 작은 부분부터 확인한다.
