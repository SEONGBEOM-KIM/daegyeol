# 10일 AI 학습 회고

## 1. 사용자가 AI에게 잘 전달한 정보

- 코딩과 공부를 AI 활용의 핵심 목표로 정했다.
- 웹앱을 배포까지 완성하고 싶다고 밝혔다.
- 속도보다 직접 이해하며 배우는 것을 우선한다고 정했다.
- 작은 단위로 함께 작성하는 방식을 선호한다고 설명했다.
- 중요한 선택은 대안과 장단점을 보고 함께 결정하고 싶다고 했다.
- `대결` 프로젝트의 문제, 사용자, 핵심 흐름, 제외 범위를 점진적으로 좁혔다.
- Next.js와 Supabase를 직접 선택하고 그 이유를 학습 목표와 연결했다.

## 2. 모호해서 다시 질문이 필요했던 부분

- 처음에는 만들고 싶은 앱 주제가 여러 번 바뀌었다.
  - 학습 도우미 앱
  - 자녀와 갈 장소 기록 앱
  - 선택을 대신 도와주는 앱
- “모든 분야를 추천하고 싶다”는 요구는 구현 범위가 커서 공통 입력 방식으로 줄여야 했다.
- 로그인, 외부 데이터 연동, AI API 연결 시점은 실제 구현 전에 다시 결정해야 한다.
- 추천 품질을 어떻게 평가할지는 채택률 외에 더 구체화할 여지가 있다.

## 3. 자동화하면 좋은 반복 작업

- 첫 화면처럼 각 기능별 자동 검증 스크립트 만들기
- 기능별 수동 체크리스트 만들기
- `npm run build`를 매 작업 후 실행하기
- 배포 전 환경 변수 누락 확인하기
- 비밀 키가 코드나 문서에 들어갔는지 검사하기
- PR 또는 커밋 전 변경 파일 요약 생성하기

## 4. 다음 프로젝트에서도 유지할 규칙

- 바로 코딩하지 않고 먼저 목적과 범위를 정한다.
- Must / Should / Later로 기능을 나눈다.
- 코드를 작성하기 전에 변경 파일을 설명한다.
- 작은 변경은 자동으로 진행하되, 중요한 선택은 확인받는다.
- 기능 하나가 끝나면 테스트 방법을 알려준다.
- 검증 결과를 보고한 뒤 다음 작업으로 넘어간다.
- 배포와 비밀 정보 관리를 별도 체크리스트로 다룬다.

## 5. 앞으로 공부할 주제

1. Next.js App Router와 서버/클라이언트 컴포넌트 구분
2. Supabase Auth, PostgreSQL, Row Level Security
3. OpenAI API를 서버 측에서 안전하게 호출하고 구조화된 결과를 받는 방법
4. Git과 GitHub를 활용한 변경 이력 관리
5. 자동 테스트와 CI/CD

10일 학습을 통해 AI에게 단순히 “만들어줘”라고 요청하는 방식에서 벗어나, 다음 흐름으로 협업할 수 있게 되었다.

```text
목표 정의

→ 오류 분석
```
